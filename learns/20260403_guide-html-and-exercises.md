# 2026-04-03：學習指南 HTML 化 + 實作練習素材大改造

## 我們做了什麼？

今天做了三件大事，全圍繞著同一個目標：**讓學習指南從「看的」變成「用的」**。

### 第一件：把 Markdown 變成精美 HTML 網站

用 `ui-ux-pro-max`（設計系統推薦）+ `frontend-design`（美學實作）兩個 Skill 搭配，寫了一個 Python 腳本 `build_guide_html.py`，把 6000+ 行的 Markdown 一鍵轉成漂亮的 HTML 文件網站。

**網站功能：**
- 左側摺疊式章節導航（34 章 + 子節點）
- 搜尋功能（Ctrl+K 彈出搜尋框）
- 深色/淺色模式切換
- 閱讀進度條（頂部漸層條）
- 滾動時自動顯示「目前在第幾章」的指示條
- 程式碼區塊附「複製」按鈕
- 響應式設計（手機也能看）

**設計選擇：**
- 風格：Swiss Minimalism（瑞士極簡）
- 字體：Noto Serif TC（標題）+ Noto Sans TC（內文）+ JetBrains Mono（程式碼）
- 配色：Indigo 靛藍色系
- 來自 ui-ux-pro-max 的建議，不是拍腦袋決定的

### 第二件：85 個案例從「參考資料」變成「動手做練習」

原本每個案例 ~65 行，充滿英文提示詞、Claude 產出描述、冗長的技巧說明。改成每個 ~15 行的實作練習格式：

```
#### 實作 14：合約審查標記與談判
> 這個練習：用 Claude 分析法律文件、整理重點

準備素材：exercises/ex14_vendor_contract.md

在 Claude Code 中輸入：
> 請幫我審查這份供應商服務合約...

進階挑戰：
- 標記所有對我方不利的條款
- 建議修改方向
```

### 第三件：建立 81 個練習素材檔案

為 61 個需要輸入資料的練習，生成了 81 個模擬素材檔案放在 `exercises/` 資料夾。

包括：假的供應商合約、SaaS 營收表、銀行對帳單、HubSpot CRM 匯出、Slack 頻道摘要、手寫食譜、辯論案例⋯⋯各種奇奇怪怪但很實用的檔案。

## 為什麼這樣做？

### HTML 化：方便查閱

11,000 行的 Markdown 用文字編輯器看很痛苦。變成網頁後有導航、有搜尋、有章節指示條，閱讀體驗完全不同。

### 案例實作化：從「看菜單」到「進廚房」

想像你去餐廳。

- **舊版案例**：服務生給你一本精裝菜譜，每道菜有三頁的食材來源介紹、廚師背景故事、英文翻譯⋯⋯你看得很飽，但一口都沒吃到。
- **新版練習**：服務生直接把食材放你面前，給你一張 A4 食譜說「照這個做」。你 15 分鐘後就端出一盤菜。

### 素材檔案：別讓同學卡在「沒有資料」

很多練習的提示詞會說「請分析我的 CRM 資料」、「請審查這份合約」。但同學手邊沒有 CRM、沒有合約啊！

提供模擬素材 = 給同學一個安全的沙盒，不用擔心弄壞自己的真實資料，也不用花時間準備測試檔案。

## 學到的小知識

### 1. 自己寫 Markdown 解析器要注意的坑

寫了一個 `build_guide_html.py` 裡面有完整的 Markdown → HTML 解析器。踩了兩個大坑：

| 坑 | 原因 | 修法 |
|---|---|---|
| 表格空欄位消失 | `split('|')` 後用 `if c.strip()` 過濾，空格子被判定為假值 | 改成只去頭尾空字串，保留中間的空欄位 |
| 換行符失效 | Markdown 的 `兩個空格+換行` 表示 `<br>`，但 `line.rstrip()` 先把空格吃掉了 | 先檢查 `endswith('  ')`，再 rstrip |

**記住**：Markdown 裡「兩個空格」是有意義的！不要隨便 strip。

### 2. CSV 在 Windows 上的亂碼問題

用 Python 寫 CSV，中文全變成亂碼 `���`。

**原因**：Python 預設用 UTF-8 寫檔，但 Windows 的 Excel 預設用 Big5 (CP950) 讀檔。兩邊編碼不一樣，就變亂碼。

**解法**：寫檔時用 `encoding='utf-8-sig'` 而不是 `encoding='utf-8'`。差一個 `-sig`，就是在檔案開頭加了一個 BOM（Byte Order Mark，位元組順序標記），等於在檔案最前面插了一個小標籤說「嗨我是 UTF-8 喔」，Excel 看到就懂了。

### 3. 滾動時不知道在哪 → 加「章節指示條」

長頁面最大的 UX 問題就是「我現在到底看到哪了？」。

解法是加一條固定在頂部的「章節指示條」，用 JavaScript 的 scroll spy 偵測目前可見的 `<h2>` 標題，即時更新顯示文字。往上滾回頂部時自動隱藏，不佔空間。

### 4. 「練習技能：Write + Edit」是天書

原本在每個練習標註「練習技能：Write + Edit」，以為很專業。但這是 Claude Code 的內部工具名稱，對非技術背景的同學來說完全看不懂。

改成人話：「這個練習：用 Claude 打造個人工具或網頁」。

**教訓**：寫給新手看的東西，每一個詞都要問自己「他看得懂嗎？」

### 5. 有素材的練習要「自報家門」

光是把素材檔案放在 `exercises/` 資料夾還不夠。如果練習本身沒有提到「請先準備 ex14_vendor_contract.md」，同學根本不知道有這些檔案存在。

解法：在 `integrate_cases.py` 裡掃描 `exercises/` 資料夾，自動把對應的檔案清單嵌入每個練習裡。

## 遇到的問題與解決方式

### 問題 1：滾動不知道在第幾章

**現象**：HTML 頁面很長，往下滾就迷路了。
**解決**：加了固定在頂部的章節指示條（`#chapter-bar`），偵測 `.chapter-heading` 元素的位置，即時顯示目前章節。

### 問題 2：表格排版錯亂

**現象**：「Skill vs Command 怎麼選？」表格的 ✅ 全部擠到 Command 欄。
**原因**：Markdown 表格解析器把空欄位過濾掉了。`| | ✅ |` 的空欄位被 `if c.strip()` 判定為假值而丟棄。
**解決**：重寫 `_split_row()` 方法，只去掉首尾的空字串，保留中間所有欄位（包含空的）。

### 問題 3：29 個練習缺少素材檔案

**現象**：實作 75 的提示詞說「從 HubSpot 提取輸單交易」，但沒有提供模擬資料。
**解決**：全面清查 85 個練習，發現 29 個有漏洞。寫了 `generate_missing_exercises.py` 一次補齊，包括 HubSpot 匯出、Slack 頻道摘要、行事曆、臨床試驗數據等。

### 問題 4：CSV 中文亂碼

**現象**：Windows 上打開 CSV，中文全是 `���`。
**解決**：`encoding='utf-8'` 改成 `encoding='utf-8-sig'`，加 BOM 標記。

## 未來小提醒

- `build_guide_html.py` 是自製的 Markdown 解析器，不是用現成的 markdown 套件。如果遇到新的 Markdown 語法（例如腳註、定義列表），需要手動擴充解析器
- 每次修改 `integrate_cases.py` 或 `generate_exercises.py` 後，記得跑完整流程：`restore → integrate → build_html`
- 素材檔案目前只有文字類的（CSV、MD、TXT）。如果未來需要圖片類的練習素材（例如植物辨識），需要另外準備
- HTML 檔案 835 KB，在手機瀏覽器上可能稍慢。如果未來要優化，可以考慮分頁載入或靜態站點生成器（Hugo/Astro）
- Windows 上任何要給 Excel 讀的 CSV，永遠用 `utf-8-sig`，這個教訓值一百次 debug
