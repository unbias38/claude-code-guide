# 2026-04-03：85 個使用案例融入學習指南實戰

## 我們做了什麼？

把辛苦整理的 `claude_use_cases_zh.md`（85 個 Claude 官方使用案例，5827 行）融入 `Claude-Code-學習指南-從入門到精通.md`。

關鍵字：**融入，不是丟進去。**

### 第一次嘗試（被打槍）

一股腦把 85 個案例塞進「附錄 A」，加個分類索引就交差。被自己否決了 — 這不叫融入，這叫偷懶。

### 第二次做法（正確做法）

寫了一個 Python 腳本 `integrate_cases.py`，做到真正的分散嵌入：

| 做法 | 數量 | 說明 |
|------|------|------|
| **嵌入現有章節** | 43 個案例 → 14 個章節 | 每個章節末尾加「延伸實戰案例」區塊，放的案例跟該章主題直接相關 |
| **新增行業章節** | 42 個案例 → 4 個新章節 | 第 31-34 章，按行業分類（非營利教育、商務、財務法律研究、個人生活） |

最終結果：4,399 行 → 11,217 行，85 個案例全部到齊零遺漏。

## 為什麼這樣做？

### 「附錄」vs「融入」的差別

想像你買了一套百科全書。

- **附錄做法**：把 85 張剪報全部夾在最後一頁。讀者翻到最後才看到，跟前面的內容完全斷開。
- **融入做法**：在講「水管維修」的章節旁邊放「水管案例」，講「電路」的章節旁邊放「電路案例」。讀者學到那裡，馬上就能看到實際應用。

### 為什麼還需要新增 4 個行業章節？

85 個案例中，有些是跟特定行業相關（募款、法律合約、臨床研究），不屬於任何 Claude Code 技術章節。這些案例按行業分組成獨立章節，讀者可以直接找自己的行業。

## 學到的小知識

### 1. 「融入」的三個層次

| 層次 | 做法 | 效果 |
|------|------|------|
| 最懶 | 全部丟附錄 | 有放但沒人看 |
| 中等 | 每章放連結/摘要，全文在後面 | 有引導但還是要翻 |
| 最好 | 全文直接嵌在對應章節 | 學到哪看到哪，無縫融入 |

我們選了最好的做法。

### 2. 大量文字整合的技術手段

當你要把 5800 行的內容插入一個 4400 行的檔案，而且要插到 14 個不同的位置 — 手動做會瘋掉。正確做法：

- **用 Python 腳本**自動化（`integrate_cases.py`）
- 腳本做的事：解析案例 → 辨識章節邊界 → 調整標題層級 → 自動插入正確位置
- **從後往前插入**（倒序操作），這樣前面的位置不會因為插入而偏移

### 3. Markdown 標題層級要跟著上下文調整

原始案例用 `##` 當標題。但塞進已有 `##`（章節標題）和 `###`（小節標題）的指南裡，案例標題就得降級：

```
原始：          嵌入後：
## 案例標題  →  #### 案例標題
### 小節    →  **小節**（改用粗體）
```

這樣才不會跟指南本身的結構打架。

### 4. 案例-章節映射是最花腦力的部分

85 個案例分配到 14 個章節 + 4 個新章節，每個都要想清楚：

- MCP 章節 → 放瀏覽器擴充案例（CRM、比價、行事曆）
- Skill 章節 → 放品牌素材、設計技能案例
- 進階工作流 → 放每日簡報、批量處理、專案報告
- 不適合任何技術章節的 → 按行業分組成新章節

這個映射表就是整個任務最核心的「設計」。

### 5. 讀者導航表要同步更新

在指南開頭的「你是哪種讀者？」表格加了一行：

```
| 想看 Claude 在各行業怎麼用 | 第 31-34 章（各章節內也有散落案例） | 先找你的行業分類 |
```

這樣新讀者第一眼就知道案例在哪裡。

## 遇到的問題與解決方式

### 問題 1：第一次用「附錄」做法被打槍

**為什麼錯**：使用者說「融入」，我卻做了「附加」。把 5800 行丟進附錄 ≠ 融入。

**解決**：還原到原始檔案，重新設計分散嵌入方案，用 Python 腳本自動化。

**教訓**：「加入」這個詞很含糊。當使用者說要把 A 加入 B，要先問清楚「怎麼加」— 是附加在後面，還是拆開散佈在各處？差別很大。

### 問題 2：大量案例不全都適合現有章節

85 個案例裡，有些是 Claude 網頁版（不是 Claude Code）的案例，沒辦法硬塞進 CLI 工具的技術章節。

**解決**：分兩路走 — 技術相關的嵌入現有章節，行業相關的獨立成新章節。

### 問題 3：殘留的舊修改

第一次嘗試的附錄被還原了，但導航表裡加的「附錄 A」那行沒清乾淨，差點殘留在最終版本。

**教訓**：還原操作後要全面檢查，不能只看主要區域。殘留的碎片很容易漏掉。
