# 2026-03-31 — Blog 寫作與深度調查實戰

## 我們做了什麼？

從一個簡單的問題出發：「YouTube 上那些 Claude Code 影片是真的嗎？」，做了一次完整的深度調查，最後產出了一篇可以發布在 HackMD 上的 blog 文章。

整個過程：
1. **深度調查** — 同時派出三個 AI 代理，分別搜 YouTube 趨勢、Claude Code 真實能力、vibe coding 現實落差
2. **初稿撰寫** — 把調查結果整理成有結構的文章
3. **平台選擇** — 從方格子 → 發現 Markdown 格式問題 → 改用 HackMD
4. **去 AI 味** — 用 humanizer-zh-tw skill 去除 AI 寫作痕跡
5. **避免踩雷** — 避開競品來源、避免指名道姓、避免可搜尋到特定影片的關鍵字
6. **HackMD 排版** — 加入提示框、mermaid 圖表、摺疊區塊、螢光筆等功能
7. **Perplexity 補強** — 用 Perplexity 搜更多佐證資料，再次跑 humanizer
8. **標題精簡** — TOC 被截斷，把所有標題砍短

## 為什麼這樣做？

### 調查方法：平行搜尋
一個人搜一個方向太慢，也容易有偏見。同時派三個代理從不同角度搜（YouTube 生態、真實能力、現實落差），就像同時派三個記者去跑不同路線，回來再交叉比對。

### 平台選擇：方格子 vs HackMD
- **方格子**是富文本編輯器（像 Word），不吃 Markdown 語法，貼上去會顯示原始符號
- **HackMD** 原生支援 Markdown，貼上去直接能用
- 而且方格子引用 Medium 等平台的文章等於幫競品導流量，HackMD 是共筆平台，不衝突

### 去 AI 味：為什麼要跑 humanizer？
AI 寫出來的文字有固定模式（破折號太多、三段式列舉、金句收尾、粗體轟炸），讀者一眼就看得出來。humanizer 就像校稿編輯，專門抓這些痕跡，讓文章讀起來像真人寫的。

### 避免踩雷
- 不在競品平台（方格子）引用競品來源（Medium、Substack、DEV.to）
- 不用具體影片標題當例子（「11 分鐘建一個 SaaS」→ 改成概括描述），免得搜到特定創作者
- 結論焦點放在「內容與現實的落差」而不是「YouTuber 在蹭流量」，講事實不講動機

## 學到的小知識

### HackMD 進階排版
| 語法 | 效果 | 適合用在 |
|---|---|---|
| `[TOC]` | 自動產生目錄 | 長文章開頭 |
| `:::info` `:::warning` `:::danger` `:::success` | 彩色提示框 | 強調重點、警告、數據 |
| `:::spoiler 標題` | 可摺疊區塊 | 參考資料、長列表 |
| `==文字==` | 螢光筆標記 | 關鍵數字 |
| ` ```mermaid ` | 流程圖、圓餅圖 | 視覺化概念 |
| `> **文字**` | 引用框 + 粗體 | 結尾金句 |

### HackMD 發表流程
1. 右上角「分享」按鈕 → 「發表」分頁 → 勾同意社群守則 → 「公開發表筆記」
2. 發表後無法變回私人（但可以編輯和刪除）
3. 可以設定自訂網址和預設瀏覽模式

### 寫 Blog 的注意事項
- **引用來源要注意平台競爭關係**：在 A 平台發文就別大量引用 B 競品平台的文章
- **具體標題、名字會被搜尋到**：如果不想讓特定人不舒服，用概括描述取代直接引用
- **「講事實不講動機」**：說「影片跟現實有落差」比「YouTuber 在蹭流量」安全得多
- **平衡報導**：純批評容易得罪人，加入成功案例和正面研究讓文章更有說服力

### Perplexity 的用法
- 可以一次問多個子問題，用 (1)(2)(3) 編號
- 它會附來源，但要自己判斷來源品質
- 搜完把結果丟給 Claude，讓 Claude 判斷哪些值得加進文章

### Humanizer 雙重審查流程
1. 第一輪：掃描 AI 模式（破折號、粗體、三段式、金句、自問自答等）
2. 改寫問題片段
3. 第二輪：問自己「哪裡還是一看就像 AI 寫的？」
4. 再改一次

## 遇到的問題與解決方式

### 方格子不支援 Markdown
**問題**：貼上去還是顯示 `#`、`##` 等原始符號
**原因**：方格子用富文本編輯器，不是 Markdown 編輯器
**解決**：直接改用 HackMD，省去格式轉換的麻煩

### TOC 標題被截斷
**問題**：HackMD 側邊目錄的標題太長，顯示 `Claude Code 真的這麼神...`
**解決**：把所有標題砍到 6-8 個字以內（「安全的問題比你想的嚴重」→「安全問題」）

### 新增段落沒過 humanizer
**問題**：用 Perplexity 補資料後直接加進文章，忘了再跑一次 humanizer
**教訓**：每次大幅修改內容後都要重新跑一次，不然新舊段落語氣不一致

## 未來小提醒

- **寫完文章先列一次「這篇會不會得罪誰」的清單**，比發出去才想好
- **HackMD 發表前先用私人筆記預覽排版**，確認 mermaid 圖表和提示框都正常
- **humanizer 要在最後一步跑**，不是中間跑。中間加新內容後要再跑一次
- **Perplexity + Claude 是好搭配**：Perplexity 負責搜、Claude 負責判斷和整合
